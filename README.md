# Лабораторная работа №1: Протоколы и источники данных


## Структура проекта

- **`protocol.py`** — описание контракта источника данных через `Protocol`. Определяет интерфейс `DataSource` с методом получения данных.
- **`sources.py`** — реализации конкретных источников:
  - `ListSource`: возвращает данные из статического списка.
  - `FileSource`: считывает данные из текстового файла.
  - `RandomSource`: генерирует случайные числа.
- **`processor.py`** — класс-обработчик, который принимает любой объект, соответствующий протоколу `DataSource`, и выполняет над ним операции (фильтрация, сортировка, агрегация).
- **`main.py`** — демонстрация полиморфизма: запуск обработки данных из разных источников без изменения кода процессора.

## Ключевые особенности реализации

1. **Структурная типизация (Protocol)**  
   Вместо жесткого наследования от абстрактного базового класса используется `typing.Protocol`. Класс считается совместимым с интерфейсом, если реализует требуемые методы, независимо от иерархии наследования. Это обеспечивает гибкость и слабую связность компонентов.

2. **Инверсия зависимостей (DIP)**  
   Модуль обработки (`Processor`) зависит не от конкретных классов источников, а от абстракции (`DataSource`). Это позволяет легко добавлять новые типы источников (например, API или базу данных) без модификации логики процессора.

3. **Единый интерфейс доступа**  
   Все источники реализуют метод `get_data() -> List[int]`, что позволяет процессору работать с ними единообразно, используя стандартные конструкции Python (`for`, `sum`, `sorted`).

4. **Чистая архитектура**  
   Разделение ответственности: источники отвечают только за предоставление данных, процессор — за бизнес-логику. Отсутствие побочных эффектов в методах получения данных.

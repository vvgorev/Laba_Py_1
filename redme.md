# Лабораторная работа №1: Источники задач и контракты

## Цель
Реализовать подсистему приёма задач из различных источников используя duck typing и контрактное программирование (Protocol) без общего базового класса.

## Структура проекта

- **`models.py`** — структура задачи `Task` (TypedDict с полями `id: str` и `payload: Any`)
- **`contracts.py`** — протокол `TaskSource` с декоратором `@runtime_checkable` и функция `validate_source()` для runtime-проверки контракта через `isinstance()`
- **`sources.py`** — три реализации источников:
  - `FileTaskSource` — чтение задач из JSON-файла
  - `GeneratorTaskSource` — программная генерация задач
  - `ApiTaskSource` — имитация внешнего API
- **`main.py`** — точка входа, демонстрация работы со всеми источниками через единый интерфейс
- **`tasks.json`** — пример файла с задачами

## Ключевые особенности

1. **Duck Typing** — источники не связаны наследованием, но реализуют метод `get_tasks()`
2. **Protocol** — описывает контракт (требования к источнику)
3. **@runtime_checkable** — позволяет проверять типы во время выполнения
4. **Расширяемость** — можно добавить новый источник без изменения существующего кода
